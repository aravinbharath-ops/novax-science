NOVAX SCIENCE — CLOUDFLARE D1 SETUP

This version uses Cloudflare D1 for secure serial-number lookup.

1. CREATE THE DATABASE
Cloudflare Dashboard
→ Storage & Databases
→ D1 SQL Database
→ Create Database

Database name:
novax-certificates

2. RUN THE DATABASE SQL
Open the new D1 database
→ Console

Paste and run the contents of schema.sql.
Then paste and run the contents of seed.sql.

3. BIND D1 TO YOUR PAGES PROJECT
Cloudflare Dashboard
→ Workers & Pages
→ novax-science
→ Settings
→ Bindings
→ Add binding
→ D1 database

Variable name:
DB

Select:
novax-certificates

Save the binding.

4. UPLOAD THIS PROJECT TO GITHUB
Replace the current site files with the contents of this ZIP.
Make sure the functions folder is uploaded.

5. REDEPLOY
After the DB binding is added, trigger a new deployment or commit any file change.

ADDING A NEW CERTIFICATE
Upload the certificate image to:
assets/certificates/

Then run an INSERT statement in the D1 Console:

INSERT INTO certificates
(serial_number, product_name, strength, task_number, batch_number, analysis_date, measured_result, purity, certificate_url, status)
VALUES
('YOURCODE', 'Product Name', '10 mg', '123456', 'BATCH-001', '15 July 2026', '10.2 mg', '99.5%', '/assets/certificates/your-file.jpg', 'active');

REVOKING A CERTIFICATE
UPDATE certificates
SET status = 'revoked'
WHERE serial_number = 'YOURCODE';
